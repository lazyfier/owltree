# Galgame UI Correction Design

## Problem
The previous pass changed the palette but did not actually express anime / 二次元 identity. It read as a pink recolor of the same portal cards instead of a site built with visual novel UI language.

## Corrected Goal
Keep the current pages, content, and routes exactly as they are, but restyle them so they feel like a polished galgame / AVG main menu system.

## What Changes This Time

### 1. Structure Language, Not Just Color
The site should borrow from:
- route selection screens
- save/load slots
- character/profile windows
- system/config menus
- information terminal panels

This means cards need title bars, framed sections, route tags, panel rails, and menu-style states.

### 2. Page Roles
- **Home** → main menu / route select
- **Games** → chapter select / route list
- **Notes** → scenario log / archive view
- **Tools** → system menu / config list
- **Trends** → information terminal / world log

### 3. Visual Rules
- Use framed panels instead of plain frosted cards
- Use small menu labels, route labels, chapter-like headings, and slot-like rows
- Use soft cream / silver / sakura / lilac tones, but color must support the structure instead of being the structure
- Hover states should feel like “currently selected menu item”

## What Must Not Change
- No new modules or widgets
- No new routes
- No new content blocks
- No character illustrations
- No giant sticker/decal overload

## Success Criteria
- The site feels like a visual novel interface through layout grammar, framing, labels, and control styling
- The anime identity is visible even if the pink palette were removed
- Pages remain readable and useful as a personal portal
