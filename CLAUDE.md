## Project Overview

Owltree is a terminal-first personal portal built with React 18, Vite, and TypeScript.
The active runtime surface is intentionally small:

- Home terminal shell
- Recursive Markdown notes browser and reader
- Configurable frontend projects list

Legacy game/trends routes have been removed from the runtime. Static visual references may still live under `theme/` or `archive/`, but they are not mounted by the app.

## Current Runtime Structure

```text
src/
├── App.tsx                     # HashRouter shell
├── components/
│   ├── layout/                 # Footer, ShortcutHelp
│   ├── notes/                  # Markdown renderer
│   ├── portal/                 # TerminalHome, ProjectRow, mark
│   └── ui/                     # Shared host UI
├── config/                     # Project/social link env helpers
├── content/notes/              # Published Markdown source files
├── contexts/                   # Fixed terminal ThemeContext
├── data/                       # Notes/projects/index metadata
├── hooks/                      # Global shortcuts and hover card
├── lib/                        # Utility helpers
├── pages/                      # Home, Notes, NoteDetail, Projects
└── styles/                     # Terminal global and page CSS
```

## Notes

- Published notes live under `src/content/notes/`.
- Recursive folders are supported.
- Markdown detail routes use `#/notes/<folder>/<file>`.
- `src/data/contentMetadata.generated.ts` is generated by the Vite plugin from local file mtimes during dev/build.
- Do not manually edit timestamp values in application code.

## Projects

- Runtime project rows live in `src/data/projects.ts`.
- Link and visibility env vars are derived from project `id`.
- Example: `id: "owltree-portal"` maps to:
  - `VITE_PROJECT_VISIBLE_OWLTREE_PORTAL`
  - `VITE_PROJECT_LINK_OWLTREE_PORTAL`
- Empty links render as visible but non-clickable rows.

## Footer Links

Footer social links live in `src/config/socialLinks.ts`.
Optional links should stay hidden when unset; do not hardcode placeholder profile URLs in `Footer.tsx`.

## Theme And Routing

- Runtime theme is fixed to `terminal`.
- `ThemeContext` sets `data-theme="terminal"` and ignores old theme localStorage values.
- HashRouter is used for GitHub Pages compatibility.
- Keyboard shortcuts:
  - `n`: notes
  - `p`: projects
  - `Esc`: parent path
  - `?`: shortcut help

## Testing Notes

- Unit tests cover route shell, notes parsing, Markdown rendering, config helpers, and UI utilities.
- E2E tests cover the homepage modules and unknown route fallback behavior.
- Keep new behavior covered before broad cleanup.
